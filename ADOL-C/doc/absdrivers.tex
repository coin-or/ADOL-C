\subsection{Drivers for Abs-Normal Form}
\label{absdrivers}
%
In this subsection we consider functions $y=F(x):\R^n \rightarrow \R^m$ that are non-smooth because of the occurrence of 
the absolute value function. The drivers provided generate a piecewise-linear 
approximation of the function $F$ in a point $x_0$ and first order derivatives of 
this second order approximation. The piecewise-linear model will be called piecewise 
linearization in the following. Further information about the piecewise linearization 
can be found in \cite{Griewank13}.

In this manual, the piecewise linearization is written in the same order as
the corresponding driver interfaces: function outputs first and switching
variables second. Thus the \textit{abs-normal} form is given by
\[ \left[\begin{array}{c}
          y\\z
         \end{array}\right]
   =\left[\begin{array}{c}
          b\\c
         \end{array}\right]      
    +\left[\begin{array}{cc}
          Y & J\\ Z & L
         \end{array}\right] 
    \left[\begin{array}{c}
          x\\|z|
         \end{array}\right].
\]
Here $z\in\R^s$ is a vector of $s\ge 0$ \textit{switching variables} and correspondingly 
the two vectors and four matrices specifying the function $F$ have the format
\[c\in\R^s,\quad Z\in\R^{s\times n},\quad L\in\R^{s\times s},\quad b\in \R^m,\quad Y\in\R^{m\times n},\quad J\in\R^{m\times s}.\]
The matrix $L$ is strictly lower triangular.
This is simply the usual abs-normal form written in the row ordering used by
the drivers below, namely rows $[y;z]$ and columns $[x;|z|]$.

To compute the piecewise linearization $\Delta F(x_0;x)$ the abs-normal mode has to be 
enabled by using
\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf enableMinMaxUsingAbs()}
\end{tabbing}
before the $\mathit{trace\_on}()$ command.
If one is interested in the number $s$ of absolute value functions occurring in the function 
evaluation, one can get it by
\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int get\_num\_switches(tag)}\\
\>{\sf short int tag;}         \> // tape identification 
\end{tabbing}
after tracing the function $F$.

In abs-normal mode several drivers are available. For a start there is a driver 
to evaluate $y=F(x)$. The driver also returns the values of $z$ which are the arguments 
of the intermediate absolute value functions. The evaluations of the absolute value 
function is interpreted as $|z_i|=\sigma_i*z_i$ with $\sigma_i=sign(z_i)$ in the zero 
order mode. The vector $\sigma\equiv \{-1,0,1\}^s$ is called signature vector.

\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int zos\_pl\_forward(tag,m,n,keep,x,y,z)}\\
\>{\sf short int tag;}         \> // tape identification \\
\>{\sf int n;}                 \> // number of independent variables $n$\\
\>{\sf int m;}                 \> // number of dependent variables $m$\\
\>{\sf int keep;}              \> // flag for reverse mode preparation\\
\>{\sf double x[n];}           \> // independent vector $x$ \\
\>{\sf double y[m];}           \> // dependent vector $y=F(x)$ \\
\>{\sf double z[s];}           \> // argument of $\mathbf{abs}(z)$ 
\end{tabbing}
%
Additionally, there are drivers for the forward mode to evaluate first order 
derivatives of the piecewise linearization. For first order derivatives another 
signature vector generated by 
\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int firstsign($z_i$, $Z[i]$)}\\
\>{\sf double z[s];}          \> // i-th component of argument of $\mathbf{abs}(z)$ \\
\>{\sf double Z[s][p];}          \> // i-th row of first derivative $Z=z\prime (x)X$
\end{tabbing}
has to be used. \textit{firstsign(u)} of a vector $u$ is defined as the \textit{sign()} 
of the first non-vanishing component of $u$ if that exists; otherwise the value is zero. 
More detailed information can be found in \cite{Griewank13}. 

The following function evaluates the abs-smooth function $y_0 = F(x_0)$ as well as its
abs-linearization with expansionpoint $x_0$ at $x_1$, usually denoted by 
$ y_1 = \Delta f(x_0, x_1)$. The corresponding switching values are also returned. 

\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int fos\_pl\_forward(tag,m,n,x0,x1,y0,y1,z0,z1)}\\
\>{\sf short int tag;}         \> // tape identification \\
\>{\sf int m;}                 \> // number of dependent variables $m$\\
\>{\sf int n;}                 \> // number of independent variables $n$\\
\>{\sf double x0[n];}          \> // independent vector $x$ \\
\>{\sf double x1[n];}          \> // tangent vector $x1$\\
\>{\sf double y0[m];}          \> // dependent vector $y_0=F(x_0)$ \\
\>{\sf double y1[m];}          \> // first derivative $y_1=F^\prime (x_0)x_1$\\
\>{\sf double z0[s];}          \> // argument of $\mathbf{abs}(z_0)$ \\
\>{\sf double z1[s];}          \> // first derivative $z_1=z_0^\prime (x_0)x_1$
\end{tabbing}

\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int fov\_pl\_forward(tag,m,n,p,x,X,y,Y,z,Z)}\\
\>{\sf short int tag;}         \> // tape identification \\
\>{\sf int m;}                 \> // number of dependent variables $m$\\
\>{\sf int n;}                 \> // number of independent variables $n$\\
\>{\sf int p;}                 \> // number of directions\\
\>{\sf double x[n];}           \> // independent vector $x$ \\
\>{\sf double X[n][p];}        \> // tangent matrix $X$\\
\>{\sf double y[m];}           \> // dependent vector $y=F(x)$ \\
\>{\sf double Y[m][p];}        \> // first derivative matrix $Y=F^\prime (x)X$\\
\>{\sf double z[s];}           \> // argument of $\mathbf{abs}(z)$ \\
\>{\sf double Z[s][p];}        \> // first derivative matrix $Z=z^\prime (x)X$
\end{tabbing}

The reverse mode function computes Jacobian-vector product of the extended
Jacobian, i.e., 
\[ \left[\begin{array}{cc}
            Y & J\\ Z & L
         \end{array}\right]^T 
         \left[\begin{array}{cc}
            w\\ v
         \end{array}\right] =
         \left[\begin{array}{cc}
          Y^Tw + Z^Tv \\J^Tw + L^Tv
         \end{array}\right]
         \] 
Two weight vectors $\texttt{lagrange} = w$ and $\texttt{lagrangeSwitch} = v$
are used. The result are then also two vectors 
$\texttt{results} = Y^Tw + Z^Tv$ and
$\texttt{resultsSwitch} = J^Tw + L^Tv$.
The reverse mode functions are given below
\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int fos\_pl\_reverse(tag,m,n,s,lagrange,lagrangeSwitch,results,resultsSwitch)}\\
\>{\sf short int tag;}         \> // tape identification \\
\>{\sf int m;}                 \> // number of dependent variables $m$\\
\>{\sf int n;}                 \> // number of independent variables $n$\\
\>{\sf int s;}                 \> // number of switching variables\\
\>{\sf double lagrange[m];}    \> // weights/seeds for dependent variables \\
\>{\sf double lagrangeSwitch[s];} \> // weights/seeds for switching variables \\
\>{\sf double results[n];}     \> // resulting adjoint values for independent variables\\
\>{\sf double resultsSwitch[s];} \> // resulting adjoint values for switching variables \\
\end{tabbing}


\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int fov\_pl\_reverse(tag,m,n,s,p,lagrange,lagrangeSwitch,results,resultsSwitch)}\\
\>{\sf short int tag;}         \> // tape identification \\
\>{\sf int m;}                 \> // number of dependent variables $m$\\
\>{\sf int n;}                 \> // number of independent variables $n$\\
\>{\sf int s;}                 \> // number of switching variables\\
\>{\sf int p;}                 \> // number of rows\\
\>{\sf double lagrange[p, m];}      \> // weights/seeds for dependent variables \\
\>{\sf double lagrangeSwitch[p, s];}\> // weights/seeds for switching variables \\
\>{\sf double results[p, n];}       \> // resulting adjoint values for independent variables\\
\>{\sf double resultsSwitch[p, s];} \> // resulting adjoint values for switching variables \\
\end{tabbing}

To extract the coefficients of the abs-normal form directly, the \texttt{abs\_normal} function in the namespace \texttt{ADOLC} should be used. It recovers the
coefficients of the output-first form
\[
  \left[\begin{array}{c}
    y\\ z
  \end{array}\right]
  =
  \left[\begin{array}{c}
    b\\ c
  \end{array}\right]
  +
  \left[\begin{array}{cc}
    Y & J\\ Z & L
  \end{array}\right]
  \left[\begin{array}{c}
    x\\ |z|
  \end{array}\right],
\]
where \texttt{cy} stores $b$ and \texttt{cz} stores $c$.

\subsubsection{abs\_normal Function}

ADOL-C provides a container class \texttt{ADOLC::AbsNormalForm} that manages the storage for all components
automatically. It is defined in \texttt{adolc/drivers/absnormalform.h}. The
header also defines the common shape base \texttt{ANFShape} together with the
dense shape type \texttt{DenseShape}, which stores the principal dimensions of
the abs-normal form:
\begin{tabbing}
\hspace{0.2in}\={\sf short int tapeId;} \hspace{3.4in}\= \kill    % define tab position
\>{\sf struct ANFShape \{} \\
\>{\sf \};} \\
\>{\sf struct DenseShape : ANFShape \{} \\
\>{\sf \quad size\_t m;} \> // output dimension \\
\>{\sf \quad size\_t n;} \> // input dimension \\
\>{\sf \quad size\_t s;} \> // number of switches \\
\>{\sf \};}
\end{tabbing}
\begin{tabbing}
\hspace{0.2in}\={\sf short int tapeId;} \hspace{3.4in}\= \kill    % define tab position
\>{\sf struct AbsNormalForm \{} \\
\>{\sf \quad DenseShape shape;} \> // principal dimensions \\
\>{\sf \quad std::vector$<$double*$>$ Y, J, Z, L;} \> // row-pointer views defining the matrices \\
\>{\sf \quad std::vector$<$double$>$ Y\_storage, J\_storage, Z\_storage, L\_storage;} \> // owning storage\\
\>{\sf \quad std::vector$<$double$>$ y, z, cy, cz;} \> // values and constants \\
\>{\sf \};}
\end{tabbing}

The \texttt{AbsNormalForm} container can be initialized in several ways:
\begin{tabbing}
\hspace{0.4in}\={\sf short int tapeId;} \hspace{3.3in}\= \kill    % define tab position
\>{\sf AbsNormalForm anf;} \> // empty container \\
\>{\sf AbsNormalForm anf(DenseShape dims);} \> // from given principal dimensions \\
\>{\sf AbsNormalForm anf = AbsNormalForm::fromTape(short int tapeId);} \> // principal dimensions inferred from tape
\end{tabbing}

The same header also defines the concept
\texttt{ADOLC::AbsNormalFormType}. It describes the public interface of an
abs-normal form container, namely the blocks \texttt{Y}, \texttt{J},
\texttt{Z}, \texttt{L}, the vectors \texttt{y}, \texttt{z}, \texttt{cy},
\texttt{cz}, and the helper routines \texttt{empty()}, \texttt{clear()},
\texttt{dims()}, \texttt{resize(...)}, \texttt{updateCy()}, and
\texttt{updateCz()}. In particular, \texttt{dims()} may return any shape type
convertible to \texttt{ANFShape}, so other containers, such as a sparse
variant, can satisfy this concept as long as they expose the same interface.

The \texttt{AbsNormalForm} object can be computed using the following 
template driver from the \texttt{ADOLC} namespace:

\begin{tabbing}
\hspace{0.5in}\={\sf short int tapeId;} \hspace{1.8in}\= \kill    % define tab position
\>{\sf enum class UpdateConsts \{ True, False \};}\\
\>{\sf int abs\_normal$<$UpdateConsts uc = UpdateConsts::True$>$(tapeId, x, anf);}\\
\>{\sf short int tapeId;}         \> // tape identification \\
\>{\sf std::span$<$double$>$ x;} \> // contiguous view of the point of evaluation \\
\>{\sf AbsNormalForm \&anf;}   \> // container for results
\end{tabbing}

The template parameter \texttt{uc} controls whether the constant vectors 
$b$ (\texttt{cy}) and $c$ (\texttt{cz}) are automatically recomputed 
from the recovered matrices and vectors such that they satisfy 
Eq.~\eqref{absdrivers}. If \texttt{UpdateConsts::False} is passed, 
only the matrices $Y, J, Z, L$ and the values $y, z$ are recovered 
from the tape. In most code the default
\texttt{UpdateConsts::True} is the appropriate choice. The span \texttt{x}
must cover the same number of entries as the independent-variable dimension
stored in \texttt{anf.dims().n}.

The \texttt{AbsNormalForm} container provides several utility member
functions. \texttt{empty()} reports whether all principal dimensions are zero.
\texttt{clear()} resets the container and its principal dimensions to zero.
\texttt{dims()} returns the principal dimensions, and \texttt{resize(dims)} can
be used to manually adjust the internal storage with ordinary
\texttt{std::vector}-style resize semantics. The helper routines
\texttt{updateCy()} and \texttt{updateCz()} allow for manual recomputation
of the constant terms $b$ and $c$ from the current data.

\subsubsection{Legacy Interface}

For existing C, Fortran, or other low-level bindings, ADOL-C also provides a
compatibility interface that exposes the raw dense blocks directly:
\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int abs\_normal(tag,m,n,s,x,y,z,Y,J,Z,L)}\\
\>{\sf short int tag;}         \> // tape identification \\
\>{\sf int m;}                 \> // number of dependent variables $m$\\
\>{\sf int n;}                 \> // number of independent variables $n$\\
\>{\sf int s;}                 \> // number of switching variables\\
\>{\sf const double *x;}       \> // point of evaluation\\
\>{\sf double y[m];}           \> // function values $y=F(x)$\\
\>{\sf double z[s];}           \> // switching variables\\
\>{\sf double **Y;}            \> // matrix $Y \in \R^{m\times n}$\\
\>{\sf double **J;}            \> // matrix $J \in \R^{m\times s}$\\
\>{\sf double **Z;}            \> // matrix $Z \in \R^{s\times n}$\\
\>{\sf double **L;}            \> // lower triangular matrix $L \in \R^{s\times s}$\\
\end{tabbing}

This interface does not update the constant vectors $b$ and $c$ directly.
When they are needed, they can be reconstructed from the returned data via
\[
  cy = y - J|z|,
  \qquad
  cz = z - L|z|.
\]
For new user code, the C++ interface above is preferred because it manages the
storage and can update \texttt{cy} and \texttt{cz} automatically.

\subsubsection{Sparse Abs-Normal Form}

One may also compute the sparsity pattern of the extended Jacobian
matrix, ordered by rows $[y;z]$ and columns $[x;|z|]$, by using the sparse drivers
 provided in the namespace {\sf ADOLC::Sparse}.

\begin{tabbing}
\hspace{0.5in}\={\sf std::span$<$uint *$>$ \&compressedRowStorage;} \hspace{0.6in}\= \kill    % define tab position
\>{\sf int jac\_pat$<$PiecewiseLinear\{\}$>$(tag,m,n,s,x,compressedRowStorage)}\\
\>{\sf short int tag;}         \> // tape identification \\
\>{\sf int m;}                 \> // number of dependent variables $m$\\
\>{\sf int n;}                 \> // number of independent variables $n$\\
\>{\sf int s;}                 \> // number of switching variables\\
\>{\sf const double* x;}       \> // point of evaluation\\     
\>{\sf std::span$<$uint *$>$ \&compressedRowStorage;} \> // span over CRS rows of $[Y\;J;Z\;L]$ \\
\end{tabbing}

To compute the numerical nonzero entries of the same extended Jacobian in
coordinate format, one may use
\begin{tabbing}
\hspace{0.5in}\={\sf short int tag;} \hspace{1.1in}\= \kill    % define tab position
\>{\sf int sparse\_jac$<$PiecewiseLinear\{\}$>$(tag,m,n,s,repeat,}\\
\>{\sf \hspace{1.95in} x,nnz,rind,cind,values)}\\
\>{\sf short int tag;}          \> // tape identification \\
\>{\sf int m;}                  \> // number of dependent variables $m$ \\
\>{\sf int n;}                  \> // number of independent variables $n$ \\
\>{\sf int s;}                  \> // number of switching variables \\
\>{\sf int repeat;}             \> // pattern / seed reuse flag \\
\>{\sf const double *x;}        \> // point of evaluation \\
\>{\sf int *nnz;}               \> // number of nonzero entries \\
\>{\sf unsigned int **rind;}    \> // row indices \\
\>{\sf unsigned int **cind;}    \> // column indices \\
\>{\sf double **values;}        \> // numerical nonzero values \\
\end{tabbing}
Again, the rows of the recovered sparse matrix are ordered as $[y;z]$, the columns as
$[x;|z|]$, and the values correspond to the block matrix $[Y\;J;Z\;L]$.
If unmanaged memory is requested for {\sf rind}, {\sf cind}, and {\sf values},
the returned arrays must be released with {\sf delete[]}.
